# Contributing to WasteIQ

First off, thanks for taking the time to contribute! 🎉 
This project is part of a hackathon submission, but we'd love for this to continue growing.

## Ground Rules
- Be respectful and courteous.
- Please create an issue to discuss major changes before opening a pull request.

## How to Contribute
### 1. Find an Issue
Look through the [open issues](#) in our repository. If you find one that interests you, feel free to claim it by leaving a comment.

### 2. Fork the Repo and Branch Out
- Fork the repository.
- Clone locally and create a branch for your feature or bug fix:
  ```
  git checkout -b feature/your-feature-name
  ```

### 3. Make the Changes
Implement your feature or bug fix. Keep your code clean, add comments where necessary. Ensure your changes do not break existing functionality.

### 4. Create a Pull Request
- Push your changes to your fork.
- Open a Pull Request from your branch to the `main` branch of this repository.
- Describe your changes clearly in the PR description.

## Setting Up Locally
Please refer to the [Getting Started](README.md#getting-started) section in the `README` for local setup steps.
